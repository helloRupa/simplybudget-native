# Splash Screen Testing Plan

## Overview

SimplyBudget has two layers to the launch experience:

1. **Native splash screen** — the OS-level screen shown immediately when the app icon is tapped, before any JS runs. Configured via `expo-splash-screen` in `app.json`. Shows `splash-icon.png` on a `#0d1117` background.

2. **JS loading screen** — shown after the native splash hides, while SQLite data loads. Defined in `app/_layout.tsx`: `SplashScreen.preventAutoHideAsync()` keeps the native splash up until `isLoaded` is true (BudgetContext has finished reading from the database), then `SplashScreen.hideAsync()` is called and the app renders.

---

## What to Test

### 1. Native Splash Screen (OS level)

| Check | Expected |
|---|---|
| Icon is centered | `splash-icon.png` appears centered on screen |
| Background color | `#0d1117` (dark, matches app background) |
| No flash of white | Background never flickers white before the icon appears |
| No distortion | Image is not stretched — `resizeMode: contain` keeps aspect ratio |
| Dark mode | Same dark background in dark mode (configured explicitly in `app.json`) |

### 2. JS Loading Screen (`_layout.tsx`)

| Check | Expected |
|---|---|
| Transition is seamless | No visible jump or flash between native splash and JS loading screen |
| Background matches | JS loading view uses `colors.background` — should match `#0d1117` exactly |
| AppName component renders | App name is centered and readable during load |
| Hides when ready | Once SQLite data is loaded, loading screen disappears and the main app renders |
| No premature hide | App does not appear before `isLoaded` is true |

### 3. Timing & Edge Cases

| Scenario | Expected |
|---|---|
| First launch (empty database) | Splash shows briefly while DB is initialized, then app loads |
| Subsequent launches (data exists) | Splash hides quickly — DB read is fast |
| Slow device | Splash stays visible until data is ready — never shows a broken/empty UI |
| App backgrounded and foregrounded | Splash does not re-appear on resume |

## How to Test

### iOS Simulator

```bash
npx expo run:ios
```

1. Wait for the app to build and launch.
2. Once running, press **Home** to background the app.
3. In Simulator: **Device → Erase All Content and Settings** to simulate a fresh install (tests first-launch DB init).
4. Rebuild and relaunch to observe the full splash sequence.

To force-observe the native splash: in Xcode, add a short artificial delay in the JS bundle load (or slow the simulator CPU via **Debug → Slow Animations** — Cmd+T).

### Android Emulator

```bash
npx expo run:android
```

Same steps as iOS. Use **Extended Controls → Cellular → No service** to simulate a slow environment if needed.

### Physical Device (requires EAS build or `npx expo run` with device connected)

- Connect device via USB with developer mode enabled.
- Run `npx expo run:ios --device` or `npx expo run:android --device`.
- Cold-launch by fully closing the app from the app switcher before each test.

---

## Platform-Specific Notes

### iOS

- The native splash is displayed by the OS using the storyboard generated by `expo-splash-screen`. It appears before any React Native code runs.
- `SplashScreen.preventAutoHideAsync()` is called at module level in `_layout.tsx`, ensuring the native splash stays up while the JS bundle loads.
- Test on both light and dark mode (Settings → Display & Brightness).

### Android

- The splash is rendered as a drawable (`splashscreen_logo.png` exists in `android/app/src/main/res/drawable-*` density buckets).
- `edgeToEdgeEnabled: true` is set in `app.json` — verify the splash extends to screen edges with no system bar artifacts.
- Test with the back gesture disabled (`predictiveBackGestureEnabled: false`) — confirm it has no effect on the splash.
- Test on a range of screen densities: mdpi, hdpi, xhdpi, xxhdpi, xxxhdpi.

## Pass Criteria

- [ ] Native splash background is `#0d1117` on both platforms
- [ ] No white flash at any point during launch
- [ ] `splash-icon.png` is centered and undistorted
- [ ] Transition from native splash to JS loading screen is seamless (no visible seam)
- [ ] JS loading screen background matches native splash background exactly
- [ ] App does not render main UI before `isLoaded` is true
- [ ] Splash does not reappear when the app is foregrounded from the background
- [ ] Tested on both light and dark mode (iOS)
- [ ] Tested on multiple screen densities (Android)
